# Arondo v0.2.5

**Release Date:** 2026-07-09
**Commits:** 4
**Tag:** v0.2.5

Authentication hardening release moving runner connections to individually issued, revocable per-runner tokens, plus a per-runner toggle for Global Agent Rules sync and a small usability fix for the `@` file selector.

---

## Highlights

- **Per-Runner Authentication Tokens**: Replaced the single shared runner access token with individually generated tokens managed alongside client tokens in `tokens.json`. Tokens are created by an admin in Settings, sent only via the `x-runner-token` header (no longer accepted as a URL query param), and compared using a constant-time check. Each token locks to the first runner identity that registers with it, so a leaked token can't be replayed to impersonate a different runner; revoking a token disconnects its runner immediately.
- **Runner Tokens Settings UI**: Added a Runner Tokens management section to Settings, and renamed the client token API from `/api/auth/tokens` to `/api/auth/client-tokens` for symmetry with the new `/api/auth/runner-tokens`.
- **Registration Race Fix**: Fixed a race where the runner client's `task.status` event, sent immediately after registering, could arrive before the server finished binding the token and get rejected as out-of-order. Messages received during registration are now queued and replayed once registration completes.
- **Per-Runner Global Agent Rules Sync Toggle**: Added a checkbox for each runner under Global Agent Rules in Settings, checked by default for new runners. Unchecking a runner stops future rule syncs and removes the previously synced rules block from its `GEMINI.md`/`CLAUDE.md` via a new `rules.remove` runner method.
- **@ File Selector Navigation Fix**: The `@` file/directory selector modal can now navigate ("Go Up") past the current project root, with an "(outside project)" label shown once the browsed path leaves the project directory.
