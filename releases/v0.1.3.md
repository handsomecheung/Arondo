# Arondo v0.1.3

**Release Date:** 2026-06-27
**Commits:** 11
**Tag:** v0.1.3

Incremental release focused on terminal session persistence, runner-side interactive shell execution, agent quota limit detection, mobile touch UX improvements, and a major codebase refactoring.

---

## Highlights

- **Terminal session persistence & reattachment**: Shell terminals spawned on runners now persist when the browser is refreshed or closed. Reopening the terminal automatically reattaches to the active PTY session on the runner and replays the terminal output buffer.
- **PTY Execution on Runner**: Interactive shell terminals have been refactored to launch directly on the runner machine rather than the server, bringing terminal execution in line with other agent and script executions.
- **API Quota & Limit Warnings**: Added support for detecting Claude's session limit hits and `agy` (Antigravity CLI) quota limits. The user is now clearly informed of limit exhaustion rather than experiencing silent failures.
- **Major Code Modularization**: Successfully refactored the monolithic `app/page.tsx` (exceeding 4,000 lines) into cleaner React components and modular custom hooks (such as `useWebSocket`, `useScripts`, and `useSessionSubmit`) for improved maintainability.
- **Mobile Touch Scroll Improvements**: Fixed terminal touch scroll hijacking on mobile devices by manually managing touch gestures in xterm.js viewports.
