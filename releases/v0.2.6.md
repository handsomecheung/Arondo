# Arondo v0.2.6

**Release Date:** 2026-07-10
**Commits:** 8
**Tag:** v0.2.6

Draft sessions, session archiving, and reliability hardening release. Introduces a new Draft session workflow for staging tasks while the codebase is mid-edit, auto-archiving of idle sessions, atomic file writes to eliminate JSON corruption races, and several UX refinements.

---

## Highlights

- **Draft Sessions**: A new "Draft" session status lets you compose a task while the project has uncommitted changes or an active agent. Drafts are real sessions (same sidebar, same SessionView UI, same three-dot menu) that auto-send once no agent is running and the codebase is clean, or manually via the composer's Send button. The now-unused `ScheduleTaskModal` and `runScript` scheduled-task action have been removed.
- **Manual/Auto Send Mode for Drafts**: Draft sessions can toggle between Auto (send automatically when codebase is ready) and Manual-only modes via a three-dot menu item, backed by `POST /api/sessions/[id]/draft-trigger`. The sidebar shows "Draft" for manual-only and "Pending" for auto-send drafts.
- **Session Archiving**: Sessions idle for more than `ARONDO_SESSION_ARCHIVE_DAYS` (default 7) are automatically archived. Sessions can also be manually archived or unarchived via the three-dot menu. Archived sessions live in a separate view opened from the sidebar's three-dot menu and are read-only (chat input disabled). File paths, logs, and diffs continue to work for archived sessions.
- **Atomic JSON File Writes / Concurrent Write Fix**: All files mutated by the server (`sessions`, `messages`, `tokens`, `agent-commands`, etc.) now go through `lib/fileLock.ts`, which serialises writes per path and atomically renames a temp file into place. This eliminates a race where concurrent callers could interleave writes and corrupt JSON files or silently drop updates.
- **Runner Display Name from Token**: Removed the runner `--name` CLI flag. The runner's display name now comes solely from the bound token's `name` field configured in Settings. The token's `boundRunnerId` is reused as the stable identity across reconnects, and renaming the token in Settings updates the live runner name immediately.
- **User Message Cards with Copy Action**: User chat messages now render as `UserMessageCard` components with the same light-purple background as `UserAgentCommandCard`, and a three-dot menu offering a Copy action.
- **Script Submenu Direction & Menu Reorder**: The Run Script submenu now opens to the left instead of downward to avoid overflow on narrow viewports, with a `useLayoutEffect`-based shift correction for edge cases. The "Archived Sessions" item has been moved to the top of the sidebar's three-dot dropdown.
- **Weekly Reset Time Parsing Fix**: Fixed a parsing bug where Claude Code's usage screen sometimes formats cross-day resets as "Jul 15 at 4:59am" instead of "Jul 1, 5am", leaving `WeekResetsAt` unset and breaking quota-based auto-selection.
