# Arondo v0.1.8

**Release Date:** 2026-07-02
**Commits:** 10
**Tag:** v0.1.8

Incremental release focused on rich UI rendering, enhanced chat input interactions for custom scripts/shell commands, fine-grained command tracking, mobile UX improvements, and desktop styling bug fixes. Key changes include rendering agent output in Markdown with syntax highlighting and clickable file links, introducing the `!` quick-run trigger in chat input to execute custom scripts or raw shell commands, adding timeline representation for user agent commands, and fixing desktop-specific duplicate file selector panel issues.

---

## Highlights

- **Markdown Rendering & Clickable File Links in Agent Output**: Replaced plain-text agent output with a React Markdown renderer that supports GFM and syntax highlighting. Practical file paths (like `lib/config.ts`) and `file://` URLs are validated against the runner's filesystem; clicking a verified path opens the file directly in Arondo's built-in File Browser.
- **Chat Input Script & Command Quick-Run Trigger (`!`)**: Added support for typing `!` in the chat textarea to trigger a project-scoped custom script autocomplete menu (like `/` does for agent slash commands). If the text after `!` doesn't match a predefined script, it falls back to running it directly as a raw shell command.
- **User Agent Command Timeline & Prompt Tracking**: Custom agent slash commands (e.g. `/commit`) are now rendered as dedicated blue `UserAgentCommandCard` nodes in the session timeline. Arondo now tracks both the raw input command and the resolved prompt payload independently.
- **Integrated Inline Log Streaming**: Refactored execution logging so that script log streams for commands triggered via `!` are displayed directly inside the `ScriptExecCard` without needing a separate quick card component.

## Features

### UI & Output Rendering
- Render agent output through react-markdown with `rehype-highlight` and GFM support (`a0ca0c4`)
- Implement verified path linkification inside markdown; clicking a link opens the file browser (`a0ca0c4`)
- Provide a "Show Raw Output"/"Show HTML" toggle option in the agent card menu (`a0ca0c4`)
- Filter stray U+FFFD replacement characters emitted by PTY split sequences to prevent rendering mojibake (`a0ca0c4`)
- Render user agent slash commands via a dedicated `UserAgentCommandCard` with custom blue styling in the session timeline (`db9bb87`)

### Chat Input & Script Execution
- Add project-scoped script autocomplete picker triggered by typing `!` in the chat input (`f9b3e2b`)
- Allow running arbitrary shell commands via the `!` prefix when it does not match a predefined script (`b6353af`)
- Stream script execution logs inline directly within `ScriptExecCard` to simplify UI and remove redundant cards (`bcac8f9`)
- Pass original commands and resolved prompt payloads to the message creation API for better prompt tracking (`db9bb87`)

### Mobile & Desktop Experience
- Reorder CSS declarations so that desktop media-query overrides correctly take precedence over mobile CSS, preventing duplicate File Browser panels from rendering on desktop (`5e36ce4`)
