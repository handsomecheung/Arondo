# Arondo v0.2.1

**Release Date:** 2026-07-05
**Commits:** 16
**Tag:** v0.2.1

Feature release focusing on multi-user token-based authentication, granular runner access control lists (ACL), secure WebSocket connections, terminology refinement (renaming 'node' to 'runner' globally), and a standalone Runners dashboard.

---

## Highlights

- **Multi-User Token-Based Authentication**: Implemented a comprehensive authentication system with `admin` and `user` roles. Automatically generates a secure admin access token on first startup if none is configured. Unauthenticated requests are automatically redirected to a new Login page (`app/login/page.tsx`).
- **Granular Runner Access Control**: Admins can manage access tokens and restrict access to specific runners by configuring an allowed list of user token UUIDs (`allowedUserTokenUuids`). All API endpoints are secured by verifying the caller's token role and access permissions.
- **Secure WebSocket Protocol Integration**: The client-to-server WebSocket (`/ws`) handshake is secured by negotiating and passing the authentication token via the `Sec-WebSocket-Protocol` header (`arondo-token`) instead of URL query parameters, preventing token exposure in proxy and server logs.
- **Standalone Runners Page**: Runner list, live details, agent quota usage progress bars, and disconnected runner cleanup have been moved to a standalone Runners page (`app/runners/page.tsx`), accessible via a new sidebar navigation link.
- **Terminology Refactoring**: Cleaned up codebase and UI references to rename `node` terminology to `runner` globally.
