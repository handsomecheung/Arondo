# Arondo v0.2.0

**Release Date:** 2026-07-04
**Commits:** 11
**Tag:** v0.2.0

Feature release focused on integrated inline visual diff viewing, rendering performance optimization, task persistence improvements, and revamped task lifecycle retention in the Tasks dashboard. Key changes include a responsive inline diff modal powered by diff2html, HTML rendering caches for agent cards, a 3-day task retention policy, and inline log streaming on the tasks page.

---

## Highlights

- **Inline Visual Diff Viewer Modal**: Added a responsive file diff viewer ([DiffModal](file:///mnt/coder-workspaces/private-workspace/repos/github/Arondo/components/modals/DiffModal.tsx)) powered by `diff2html`. Markdown file links in agent execution cards now display a diff button next to them if the file has git modifications. Clicking it pops open the visual diff modal directly in the workspace.
- **Rendering Caching & Lazy-loading**: Performance optimization for [AgentExecCard](file:///mnt/coder-workspaces/private-workspace/repos/github/Arondo/components/AgentExecCard.tsx). Generated markdown output is now cached on the server on first render, avoiding redundant rehype/remark parses on subsequent views. File existence and git diff checks are also lazy-loaded.
- **Tasks Dashboard Updates & Inline Logs**: Task page is updated to filter out chat-initiated background script tasks, defaults to showing only non-completed tasks, and allows inspecting script execution logs inline. Completed tasks retention is updated from 7 days to 3 days, and completed tasks within the retention window are now restored on server startup.

### Tasks & Core Lifecycle
- Update task retention period from 7 days to 3 days (`f6ed62e`, `8a68e95`)
- Filter out chat-initiated script tasks from tasks page (`f9d0f44`)

### Scripts & Runner
- Kill existing server and runner processes in run scripts before starting new instances to prevent conflicts (`822eae0`)
- Replace runner `fs.exists` command with `fs.infos` to support batch path exists and diff checking (`98529ac`)
