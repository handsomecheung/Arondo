# Arondo v0.2.2

**Release Date:** 2026-07-06
**Commits:** 16
**Tag:** v0.2.2

Stability and integration release introducing AI agent session continuity for Antigravity (agy), migrating auto-script discovery to remote runners, clean removal of local PTY dependencies (node-pty), and a robust Playwright integration test suite covering automode transitions and cross-agent context handoff.

---

## Highlights

- **AI Agent Session Continuity (Session Resume) for Antigravity**: Extends the session resume capabilities to Antigravity CLI (`agy`). The runner detects the newly created conversation ID from the CLI logs and maps it to the current session, enabling dialog continuity by automatically passing the `--conversation` argument in subsequent turns.
- **Server-Side PTY Removal & Dependency Cleanup**: Cleaned up deprecated local server-side PTY code ([lib/pty-manager.ts](file:///mnt/coder-workspaces/private-workspace/repos/github/Arondo/lib/pty-manager.ts)) and uninstalled `"node-pty"` from Next.js server dependencies, reducing host build dependencies and simplifying multi-machine deployments. All shell terminals now run natively on connected runners.
- **Remote Auto-Script Discovery**: Refactored the AI-powered project script detection to execute remotely on the selected runner using the `exec.agent` API, resolving local filesystem assumptions on the server and aligning with the "All execution goes through a Runner" architecture.
- **Comprehensive Playwright Integration Tests**: Introduced an end-to-end testing suite validating runner file systems, git operations, API authentications, and complex AI Agent quota states, resume behaviors, and cross-agent transition flows.
