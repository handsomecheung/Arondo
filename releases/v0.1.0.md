# Arondo v0.1.0

**Release Date:** 2026-06-25
**Commits:** 59
**Tag:** v0.1.0

This is the first release of Arondo — a mobile-first developer workspace that delegates coding tasks to AI agents and monitors executions across multiple machines.

---

## Highlights

- **Multi-machine architecture**: A central Next.js server coordinates lightweight Go-based Runners installed on any development machine, connected via WebSocket.
- **AI agent integration**: Supports Antigravity CLI (agy), Claude Code, and Codex as coding agents, with auto-detection of installed agents on each Runner.
- **Interactive PTY terminal**: Full pseudo-terminal support via Go's `creack/pty` and `xterm.js` in the browser, with ANSI colors, cursor control, and interactive stdin.
- **Mobile-first UI**: Collapsible panels, modal logs, responsive menus, and touch-friendly actions designed for use on phones and tablets.

## Architecture

```
Browser (Next.js UI)  <--ws-->  Server (Next.js)  <--ws-->  Runner A (Go, machine-1)
                                                  <--ws-->  Runner B (Go, machine-2)
```

- **Runner** (`runner/`): Go binary connecting to the Server via WebSocket. Executes commands, manages PTY sessions, runs git/filesystem operations.
- **Server**: Next.js app routing operations to Runners. Manages persistent state (sessions, projects, messages, logs).
- **Frontend**: Single-page React UI with Runner selection, file browsing, chat, terminal modals, and task queue.

## Getting Started

```bash
# Server
npm install
npm run dev

# Runner (on any development machine)
cd runner
go build -o arondo-runner .
./arondo-runner --server ws://localhost:3251/runner --name my-dev-machine
```

Open [http://localhost:3251](http://localhost:3251) in your browser.
