# Arondo v0.2.12

**Release Date:** 2026-07-24
**Commits:** 9
**Tag:** v0.2.12

PWA installability (manifest + service worker), a unified Todo Messages model replacing the old scheduled-tasks system, and a batch of mobile/runner bug fixes. This version makes Arondo installable as a standalone app on Android Chrome, collapses drafts/auto-queue follow-ups/quota retries into a single `user-todo` message concept surfaced in the Tasks dashboard, and fixes several rough edges around project readiness checks, the runner directory browser, and long repo paths on mobile.

---

## Highlights

- **PWA Installability**: Added a web app manifest (`app/manifest.ts`) with icons, and registered a service worker (`public/sw.js`) on client init — required by Chrome on Android for full standalone-window installability, not just a home-screen shortcut.
- **Unified Todo Messages**: Replaced the global `scheduled-tasks.json` store and the `draft` session status with a single `user-todo` chat message type carrying a `todoStatus`/`todoTrigger`. Drafts, auto-send-once-ready sessions, quota-exhausted retries, and follow-ups queued behind a running agent are now all the same concept, rendered as `UserTodoMessageCard` with a Cancel / Send Now / Change Trigger menu, and surfaced as first-class tasks in the Tasks dashboard.
- **Project Readiness Fix**: A running script no longer counts as "busy" when checking if a project is ready — only an actively running agent does. Fixes both the Todo Message `codebaseReady` auto-trigger and the New Session pre-send confirmation modal.
