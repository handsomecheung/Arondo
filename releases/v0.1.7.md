# Arondo v0.1.7

**Release Date:** 2026-07-02
**Commits:** 14
**Tag:** v0.1.7

Incremental release focused on visual styling transition, platform infrastructure updates, enhanced runner node management, task lifecycle reliability, and data safety. Key changes include switching the UI to light mode, centralizing configuration path resolution under a standardized user-home directory, implementing disconnected runner deletion, and adding automated data purging for orphaned sessions and projects.

---

## Highlights

- **UI Transition to Light Theme**: Re-styled the entire application from dark mode to a premium light mode. Adjusted color schemes for components, inline HTML diffs, syntax highlighting, and PTY terminal instances (`Terminal` and `ShellTerminal`).
- **Standardized Config Directory**: Shifted the default workspace configuration and state directory from local `./data` to the user home directory (`~/.arondo`), customizable via `ARONDO_CONFIG_DIR`.
- **Runner Lifecycle Management**: Added support for deleting offline or disconnected runner nodes from the Settings dashboard, along with confirmation dialogs to prevent accidental removals.
- **Orphan Data Purging**: Automated the cleanup of session and project records whose parents (associated runner or project) are no longer registered or accessible.

---

## Features

### Configuration & Data Lifecycle

- Centralize configuration directory resolution into `~/.arondo` and support `ARONDO_CONFIG_DIR` override (`79d0e57`)
- Auto-delete orphaned sessions and projects whose corresponding runner or project no longer exists (`1f96629`)
- Store global project task execution logs inside `~/.arondo/projects/[projectId]/logs/[taskId].log` for consistency (`2248a0c`)

### Runner Management

- Add the ability to delete disconnected runners from the Settings dashboard and `/api/runners` endpoint (`32153db`)
- Automatically prefer online runners on loading and disable chat input components when the active runner is offline (`9a207b9`)

### UI & Styling

- Switch global CSS variables, highlight.js, and terminal styling from dark to light mode (`50af59d`)
- Display agent type labels on the tasks page and show `script-running` states as active in the sidebar (`624feaf`)
- Implement optimistic UI updates when stopping active tasks to improve responsiveness (`89c9377`)
- Add confirmation modal dialog when deleting custom agent slash commands under Settings (`40ff1a5`)
- Polish agent selection dropdown styles to match the new light theme (`ac99636`)

## Bug Fixes

- **Runner Process Groups**: Use process groups to kill spawned tasks and prevent fork/exec `EPERM` errors on runner nodes (`31bd427`)
- **WebSocket Synchronization**: Fix task status reconciliation by ensuring `task.status` is always transmitted from runners even when no tasks are active (`e544afc`)
- **Startup Loading**: Ensure environment variables are loaded before importing submodules in `server.ts` to prevent runtime configuration failures (`464ad0c`)
